# [IT직종] Spring AI 프로그래밍 응용 (개발자 대상 AI모델 활용)

## 1. 과정 개요

본 과정은 백엔드 개발자가 **Spring AI**를 활용하여 단순 질의응답 수준을 넘어, **실서비스에 적용 가능한 AI 기능**을 설계·구현할 수 있도록 하는 응용 중심 교육입니다. **Tool Calling**, **MCP(Model Context Protocol)**, **Structured Output Converter**, **Multimodality(이미지 등)**, **Chat Memory** 등 핵심 주제를 순차적으로 다루며, 각 기술이 실제 비즈니스 요구사항(주문 조회, 규격화된 응답, 세션 유지, 외부 도구 연동)에 어떻게 대응하는지를 중심으로 학습합니다.

## 2. 필요성

생성형 AI 도입이 확산되면서 백엔드 개발자에게는 “프롬프트만 잘 쓰는 것”을 넘어, **안전하고 재현 가능한 방식으로 모델을 시스템에 녹여 내는 역량**이 요구됩니다. 모델은 본질적으로 상태가 없고(stateless), 내부 DB·레거시 API와 직접 연결되지 않으며, 응답 형식도 일정하지 않을 수 있습니다. 이를 보완하려면 **도구 호출로 사실 데이터를 가져오고**, **구조화된 출력으로 API 계약을 맞추고**, **대화 메모리로 사용자 경험을 유지하며**, 필요 시 **멀티모달 입력**과 **MCP를 통한 표준화된 외부 도구 연동**까지 고려해야 합니다. 본 과정은 이러한 실무 과제를 Spring 생태계에서 일관되게 해결하는 방법을 제시합니다.

## 3. 주요 학습 내용

| 영역 | 학습 포인트 |
| --- | --- |
| **Tool Calling** | 모델이 필요 시 Java 함수(도메인 서비스)를 호출하도록 설계하고, 스키마·설명을 통해 호출 정확도를 높이는 방법 |
| **MCP** | Model Context Protocol을 통해 외부 MCP 서버의 도구를 Spring AI와 연동하고, 운영 환경에서의 클라이언트 구성 개요 |
| **Structured Output Converter** | `entity()` 또는 `BeanOutputConverter` 등을 활용해 JSON 등 **검증 가능한 타입**으로 응답을 고정하는 패턴 |
| **Multimodality** | 이미지 URL·미디어 입력을 포함한 사용자 메시지 구성 및 비전 모델과의 연동 흐름 |
| **Chat Memory** | `MessageChatMemoryAdvisor`와 대화 ID를 활용한 **세션 단위 맥락 유지**, 윈도우 크기 등 운영 파라미터 이해 |

실습 프로젝트에서는 위 요소를 REST API 예제로 묶어, “챗봇 데모”가 아닌 **백엔드가 제공하는 AI 기능 모듈** 관점에서 구조를 잡습니다.

## 4. 기대 효과

- 사내·클라우드 LLM API와 Spring Boot 애플리케이션을 **동일한 패턴**으로 통합할 수 있습니다.
- RAG만이 아닌 **도구 실행·외부 프로토콜(MCP)·구조화 출력**까지 아우르는 설계 시야를 갖춥니다.
- API 응답 형식 불안정 문제를 **스키마 기반 출력**으로 완화하여 프론트엔드·배치·모바일과의 연동 비용을 줄입니다.
- 운영 관점에서 **메모리 정책·타임아웃·MCP 활성화 여부** 등을 설정으로 제어하는 경험을 쌓습니다.

## 5. 수강 대상

- **백엔드 개발자**로서 Spring Web 기반 서비스 개발 경험이 있으며, **중급 이상** 수준에서 Spring 생태계(의존성 주입, REST, 설정 프로파일)에 익숙한 분
- **Spring AI 기초**(ChatClient, 프롬프트 구성, 기본 모델 연동)를 학습했거나 유사 경험이 있는 분
- 단순 데모가 아닌 **실제 서비스에 투입 가능한 AI 기능**(주문 조회, 요약·분류 결과의 구조화, 세션형 상담 등)을 구현하고자 하는 분
