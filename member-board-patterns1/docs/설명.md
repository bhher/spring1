# member-board-patterns1 — 2단계: 생성자 패턴

`member-board-patterns`(1단계 Setter)를 복사한 프로젝트입니다. **DTO ↔ Entity 변환에 생성자**를 사용합니다.

## 핵심 코드 위치

| 구분 | 파일 | 내용 |
|------|------|------|
| DTO → Entity | `UserService.register` | `new User(email, hash, nickname, now)` |
| Entity → DTO | `UserResponseDto`, `BoardResponseDto` | `new UserResponseDto(user)`, `new BoardResponseDto(board)` |
| 게시글 수정 | `Board.edit(...)` | 기존 엔티티는 setter 나열 대신 도메인 메서드로 갱신 |

엔티티는 JPA용 `protected` 무인자 생성자 + `@Access(AccessType.FIELD)` 로 id 주입을 허용합니다.

## 실행

```text
cd member-board-patterns1
.\gradlew.bat bootRun
```

## 비교 학습

- **1단계**(원본 `member-board-patterns`): `new` 후 `setXxx` 연속 호출  
- **본 프로젝트**: 신규·응답 조립을 **생성자 한 번**으로 표현해 필수 필드 누락을 줄임  

다음: `member-board-patterns2`(Builder), `member-board-patterns3`(Mapper).
