# member-board-patterns3 — 4단계: Mapper 클래스 분리

변환 로직을 **`UserMapper`**, **`BoardMapper`** `@Component` 로 분리했습니다. `UserService` / `BoardService` 는 저장·검증·트랜잭션 흐름에 집중합니다.

## 핵심 코드 위치

| 매퍼 | 메서드 | 역할 |
|------|--------|------|
| `UserMapper` | `toNewEntity(dto, encodedPassword)` | 회원가입 시 `User` 생성 |
| `UserMapper` | `toResponseDto(user)` | 프로필 응답 DTO |
| `BoardMapper` | `toNewEntity(dto, author, at)` | 글 등록 시 `Board` 생성 |
| `BoardMapper` | `toResponseDto(board)` | 목록·상세 응답 DTO |
| `BoardMapper` | `apply(board, dto, updatedAt)` | 수정 시 `Board.edit` 위임 |

엔티티·응답 DTO 생성은 2단계와 동일하게 **생성자**를 사용하고, **어디서 호출하느냐**를 Mapper 로 모은 형태입니다.

## 실행

```text
cd member-board-patterns3
.\gradlew.bat bootRun
```
