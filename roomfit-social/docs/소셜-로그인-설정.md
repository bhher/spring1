# RoomFit 소셜 로그인 (Google · Naver · Kakao)

`shopping-mall1` OAuth2 패턴을 roomfit에 적용했습니다.  
(`board-login-img-security`에는 OAuth 코드가 없습니다.)

## 실행 전 준비

각 플랫폼 개발자 콘솔에서 **Redirect URI** 등록:

| 제공자 | Redirect URI (로컬) |
|--------|---------------------|
| Google | `http://localhost:8083/login/oauth2/code/google` |
| Naver | `http://localhost:8083/login/oauth2/code/naver` |
| Kakao | `http://localhost:8083/login/oauth2/code/kakao` |

dev 프로필(8084) 사용 시 포트를 `8084`로 바꿉니다.

## 환경 변수 (또는 application.yml 직접 수정)

```text
GOOGLE_CLIENT_ID=...
GOOGLE_CLIENT_SECRET=...
NAVER_CLIENT_ID=...
NAVER_CLIENT_SECRET=...
KAKAO_CLIENT_ID=...
KAKAO_CLIENT_SECRET=...   # 카카오는 비워 둘 수 있음
```

기본값 `change-me`이면 소셜 로그인은 동작하지 않습니다.

## 동작

1. `/login` → Google / 네이버 / 카카오 버튼
2. 콜백 → `CustomOAuth2UserService` → `MemberService.registerOrUpdateOAuthMember`
3. 기존 (provider+subject) → 프로필 갱신
4. 동일 이메일 일반 회원 → OAuth 연동
5. 없으면 신규 회원 (`loginId` = `google_xxx` 형식, 랜덤 비밀번호)

## 주요 파일

- `security/CustomOAuth2UserService.java`
- `security/LoginMember.java` — 폼·소셜 공통 Principal
- `config/SecurityConfig.java` — `oauth2Login`
- `application.yml` — `spring.security.oauth2.client`
