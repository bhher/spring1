# RoomFit 추천 점수 계산법

> 구현: `RecommendEngine.java`  
> 결과 화면: `/recommend/result`

추천 결과는 **프로필 매칭**, **유사 사용자(협업 필터)**, **인기글** 세 가지 점수를 각각 계산한 뒤 병합합니다.

---

## 1. 전체 흐름

```
1. 프로필 매칭  → 상위 10건 (점수 > 0)
2. 유사 사용자  → 상위 5건
3. 인기글       → 5건
4. 병합         → 중복 제거 후 최대 15건
```

병합 순서: **프로필 매칭 → 유사 사용자 → 인기글** (이미 포함된 글은 건너뜀)

---

## 2. 프로필 매칭 점수

사용자 `UserProfile`과 `InteriorPost`를 비교해 **항목별 점수를 합산**합니다.

| 항목 | 조건 | 점수 |
|------|------|------|
| 스타일 | 게시글 스타일 = 선호 스타일 | +40 |
| 평수 | \|게시글 평수 − 내 평수\| ≤ 2 | +25 |
| 평수 | \|게시글 평수 − 내 평수\| ≤ 5 | +10 |
| 예산 | \|게시글 예산 − 내 예산\| / 내 예산 ≤ 20% | +20 |
| 예산 | \|게시글 예산 − 내 예산\| / 내 예산 ≤ 50% | +5 |
| 가구 보유 | `hasFurnitureTag` 일치 | +10 |
| 좋아요 보정 | `min(5, log(좋아요수 + 1))` | 최대 +5 |

**이론적 최대 점수:** 40 + 25 + 20 + 10 + 5 = **100점**

### 계산 예시

| 프로필 | 게시글 | 계산 | 점수 |
|--------|--------|------|------|
| 미니멀, 8.5평, 150만, 가구 없음 | 미니멀, 8평, 140만, 가구 없음, 좋아요 12 | 40 + 25 + 20 + 10 + 2 | **97** |
| 북유럽, 12평, 200만 | 모던, 10평, 180만 | 10 + 5 | **15** |

> 평수·예산 조건은 **둘 중 하나만** 해당됩니다 (≤2 vs ≤5, ≤20% vs ≤50%).

---

## 3. 유사 사용자 점수 (협업 필터)

비슷한 프로필 사용자가 좋아요한 글을 추천합니다.

### 3-1. 프로필 벡터 (코사인 유사도)

```
벡터 = [평수/30, 예산/1000, 스타일 ordinal, 가구보유(0|1)]
```

두 사용자 프로필 벡터 간 **코사인 유사도**를 계산합니다.

### 3-2. 이웃 선정

- 본인을 제외한 회원 중 코사인 유사도 **> 0.3** 인 상위 **5명**

### 3-3. 글별 점수

```
점수 = Σ (이웃 코사인유사도 × 2)   // 해당 글에 좋아요한 이웃만 합산
```

- 본인이 이미 좋아요한 글은 제외
- 소수점은 **정수로 변환** (`intValue()`)

### 계산 예시

| 이웃 | 유사도 | 좋아요 여부 |
|------|--------|-------------|
| user2 | 0.8 | O |
| user3 | 0.5 | O |
| user4 | 0.4 | X |

```
점수 = (0.8 × 2) + (0.5 × 2) = 2.6 → 2점
```

---

## 4. 인기글 점수

```
base = 조회수 × 0.1 + 좋아요 × 2.0 + 댓글 × 1.5
decay = 1 / (1 + 경과일 / 7)
점수 = int(base × decay)
```

| 항목 | 가중치 |
|------|--------|
| 조회수 | × 0.1 |
| 좋아요 | × 2.0 |
| 댓글 | × 1.5 |
| 시간 감쇠 | 경과 7일마다 절반 수준으로 감소 |

### 계산 예시

조회 80, 좋아요 12, 댓글 3, 게시 후 14일 경과:

```
base = 80×0.1 + 12×2 + 3×1.5 = 8 + 24 + 4.5 = 36.5
decay = 1 / (1 + 14/7) = 1/3 ≈ 0.33
점수 = int(36.5 × 0.33) = 12
```

## 5. 결과 병합

| 단계 | 소스 | 최대 건수 |
|------|------|-----------|
| 1 | 프로필 매칭 | 10 |
| 2 | 유사 사용자 | 5 |
| 3 | 인기글 | 5 |
| 합계 | 중복 제거 | **15** |

각 게시글은 `ScoredPostDto(post, score)` 형태로 화면에 **총점**만 표시됩니다.
