# RoomFit 파이널 프로젝트 설계서

> **구현 버전**: Spring Boot 3 + Thymeleaf + JPA + Spring Security  
> 원 기획(MVC2 + JSP + Servlet + JDBC)은 동일한 3계층 역할로 **Spring MVC**에 매핑하여 `roomfit/` 모듈에 구현했습니다.

| 항목 | 내용 |
|------|------|
| 프로젝트명 | RoomFit — 1인가구 원룸 인테리어 All-in-One |
| 구현 경로 | `spring1/roomfit` (Gradle 모듈) |
| 실행 포트 | **8081** (기본 `mysql` 프로필) · **8082** (`dev` 프로필, H2) |
| DB | MySQL(기본) 또는 H2(`--spring.profiles.active=dev`) |
| 업로드 | `${user.home}/roomfit-uploads/` → `/uploads/**` 로 제공 |

---

## 1. 프로젝트 개요

1인 가구·원룸 거주자를 위한 **인테리어 정보 공유 + 맞춤 추천 + 소품 쇼핑 + 커뮤니티** 통합 플랫폼.

## 2. 기획 의도

- 분산된 인테리어 정보를 한곳에 모음
- 평수·예산·스타일 기반 **규칙+가중치+협업필터링(Lite)** 추천
- 커뮤니티·좋아요 데이터로 개인화 고도화 기반 마련

## 3. 핵심 기능

| 영역 | 기능 | 구현 클래스(예) |
|------|------|-----------------|
| 회원 | 가입/로그인/찾기/마이페이지/탈퇴 | `AuthController`, `MemberController`, `MemberService` |
| 프로필 | 추천 설문(평수·예산·스타일) | `MemberController`, `UserProfileService` |
| 인테리어 | CRUD·이미지·조회수·좋아요·댓글 | `InteriorController`, `InteriorPostService`, `FileStorageService` |
| 추천 | 프로필·색상·배치·CF·인기·소품 연계 | `RecommendController`, `RecommendService`, `RecommendEngine` |
| 쇼핑 | 상품 목록·장바구니·찜·리뷰 | `ShopController`, `ShopService` |
| 커뮤니티 | 자유/QNA/후기·신고 | `CommunityController`, `CommunityService`, `ReportService` |
| 관리자 | 통계·신고 처리·**상품 등록/수정/판매 중지** | `AdminController`, `AdminService`, `AdminProductController`, `ProductAdminService` |

## 4. 시스템 구조

```
Browser → Thymeleaf(templates/) ↔ Controller(web) ↔ Service ↔ Repository(JPA) ↔ MySQL/H2
                              ↓                        ↓
                        RecommendEngine          FileStorageService
                        ColorPaletteTable        (이미지 로컬 저장)
```

- **Controller**: HTTP·검증·Model·뷰 선택
- **Service**: 트랜잭션·비즈니스 규칙
- **RecommendEngine** (`recommend/`): 추천 알고리즘 전용 컴포넌트
- **GlobalControllerAdvice**: `BusinessException` 등 공통 예외 → flash 메시지

## 5. MVC 패턴 (Spring MVC ↔ MVC2 대응)

| MVC2 | Spring MVC (본 프로젝트) |
|------|--------------------------|
| JSP View | Thymeleaf `templates/` |
| Servlet Controller | `@Controller` |
| DAO + JDBC | `JpaRepository` |
| JavaBean Model | `@Entity` + DTO + `Service` |

## 7. 테이블 설계

JPA `ddl-auto`로 스키마 생성·갱신:

| 프로필 | `ddl-auto` | 비고 |
|--------|------------|------|
| `mysql` (기본) | `create` | 기동 시 스키마 재생성 + `DataInitializer` 시드 |
| `dev` (H2) | `update` | 개발용 in-memory |

공통: `spring.jpa.open-in-view: false` (Lazy 로딩은 Service·`@EntityGraph`에서 처리)

주요 테이블:

- `members`, `user_profiles`
- `interior_posts`, `post_images`, `post_likes`, `comments`
- `products`, `carts`, `cart_items`, `wishlists`, `product_reviews`
- `community_posts`, `reports`

상세 컬럼은 `com.example.roomfit.domain` 엔티티를 참고하세요.

## 9. 추천 알고리즘

구현: `RecommendEngine.java`  
상세 문서: [추천-엔진.md](./추천-엔진.md) · [추천-점수-계산.md](./추천-점수-계산.md)

하이브리드 3종을 병합해 **최대 15건**의 인테리어 글을 추천합니다.  
`/recommend/result`에서는 `ShopService.recommendProducts()`로 **소품**도 함께 표시합니다.

### 가중치 (프로필 매칭, `calculateProfileScore`)

| 요소 | 점수 |
|------|------|
| 스타일 일치 | +40 |
| 평수 차 ≤2 | +25 |
| 평수 차 ≤5 | +10 |
| 예산 차 ≤20% | +20 |
| 예산 차 ≤50% | +5 |
| 가구 보유 일치 | +10 |
| 좋아요 log 보정 | `min(5, log(좋아요+1))` |

이론적 최대: **100점** (평수·예산 구간은 각각 하나만 적용)

### 협업 필터 (Lite)

- 프로필 벡터 코사인 유사도 → 이웃 5명 (유사도 > 0.3)
- 이웃이 좋아요한 글 중 본인 미좋아요 글, `유사도 × 2` 합산 → 상위 5건

### 인기글 (`popularityScore`)

```
base = 조회×0.1 + 좋아요×2 + 댓글×1.5
decay = 1 / (1 + 경과일/7)
```

### 의사코드

```
FUNCTION recommend(memberId):
  profile = loadProfile(memberId)  // 없으면 예외
  A = scoreByProfile(profile, candidates, limit=10)
  B = similarUserCF(memberId, limit=5)
  C = popularPosts(limit=5)
  merged = merge(A, B, C, max=15)    // 중복 제거, 프로필→CF→인기 순
  RETURN merged + colorPalette(style) + layoutAdvice(profile)
```

## 12. 예상 질문

| Q | A |
|---|---|
| 왜 JSP가 아닌 Thymeleaf? | Spring Boot 표준 뷰, Security·폼·fragment 연동 용이 |
| 추천 정확도? | 규칙+CF Lite MVP; 좋아요·프로필 데이터가 쌓일수록 CF 비중 증가 |
| 비밀번호? | BCrypt (`PasswordEncoder`) |
| 상품 삭제는? | 물리 삭제 대신 `onSale=false` (장바구니·리뷰 FK 유지) |
| LazyInitializationException? | `open-in-view: false`; Repository `@EntityGraph`·Service 트랜잭션으로 해결 |
