# RoomFit 단계별 구현 순서

> **목적**: 처음부터 하나씩 만들어 볼 때, 어떤 순서로 하면 의존 관계가 꼬이지 않는지 정리한 가이드입니다.  
> **참고**: [프로젝트-설계서.md](./프로젝트-설계서.md) · 완성본은 `roomfit/` 모듈에 이미 있습니다.

---

## 진행 방식 (권장)

| 원칙 | 설명 |
|------|------|
| **한 단계 = 한 번에 동작 확인** | Controller → Service → Repository → Entity → 화면 순으로 맞추고 `bootRun` 후 URL로 검증 |
| **아래 단계는 위 단계에 의존** | 회원·보안 없이 추천/장바구니를 만들지 않기 |
| **시드 데이터는 나중에** | `DataInitializer`는 해당 기능 엔티티가 준비된 뒤 넣기 |

각 단계 끝에 **✅ 확인** 항목을 체크한 뒤 다음으로 넘어가세요.

---

## 전체 로드맵 (한눈에)

```
0. 프로젝트 뼈대
   ↓
1. 공통 설정 · 레이아웃 · 메인
   ↓
2. 회원 (Member) + Security
   ↓
3. 추천 프로필 (UserProfile)
   ↓
4. 인테리어 게시판 (핵심 콘텐츠)
   ↓
5. 맞춤 추천 (RecommendEngine)
   ↓
6. 소품 쇼핑 (Product · Cart · Wishlist · Review)
   ↓
7. 커뮤니티 + 신고
   ↓
8. 관리자
   ↓
9. 마무리 (예외 처리 · 파일 업로드 정리 · 문서)
```

예상 기간(학습 기준): 단계당 **0.5~2일** (경험에 따라 다름)

## 0단계 — 프로젝트 뼈대

**목표**: Spring Boot 앱이 뜨고, DB·뷰 엔진이 연결된 상태

| 할 일 | 참고 파일/설정 |
|--------|----------------|
| Gradle 프로젝트 생성 (`roomfit`) | `build.gradle`, `settings.gradle` |
| 의존성: Web, Thymeleaf, JPA, Security, Validation, H2(또는 MySQL) | `build.gradle` |
| `application.yml` — 포트 **8081**, `ddl-auto: update` | `src/main/resources/application.yml` |
| `RoomFitApplication` main 클래스 | `RoomFitApplication.java` |

**✅ 확인**

- [ ] `./gradlew bootRun` → http://localhost:8081 접속
- [ ] H2 사용 시 `/h2-console` 접속 가능

---

## 1단계 — 공통 UI · 메인

**목표**: 모든 화면이 공유하는 레이아웃과 홈만 보이게

| 순서 | 할 일 | 패키지/경로 |
|------|--------|-------------|
| 1-1 | `static/css/style.css` 기본 스타일 | `resources/static/` |
| 1-2 | `fragments/layout.html` — head, navbar, footer | `templates/fragments/` |
| 1-3 | `HomeController` — `GET /` | `web/HomeController.java` |
| 1-4 | `index.html` 메인 화면 | `templates/index.html` |
| 1-5 | (선택) `WebConfig` — 업로드 경로 `/uploads/**` | `config/WebConfig.java` |

**✅ 확인**

- [ ] `/` 에서 레이아웃·네비가 깨지지 않음

**아직 하지 말 것**: Security 전체 잠금 (다음 단계에서 점진 적용)

## 2단계 — 회원 · 로그인 (Security)

**목표**: 가입·로그인·로그아웃·역할(USER/ADMIN) 동작

| 순서 | 할 일 | 주요 클래스 |
|------|--------|-------------|
| 2-1 | Enum `Role`, `MemberStatus` | `domain/` |
| 2-2 | 엔티티 `Member` (+ `UserDetails` 구현) | `domain/Member.java` |
| 2-3 | `MemberRepository` | `repository/` |
| 2-4 | `PasswordEncoder` 빈, `SecurityConfig` URL 규칙 | `config/SecurityConfig.java` |
| 2-5 | `CustomUserDetailsService` | `config/` |
| 2-6 | `MemberService` — 회원가입, 찾기, 수정, 탈퇴 | `service/MemberService.java` |
| 2-7 | `AuthController` — 로그인·가입 화면 | `web/AuthController.java` |
| 2-8 | `MemberController` — 마이페이지, 정보 수정 | `web/MemberController.java` |
| 2-9 | Thymeleaf: `auth/login.html`, `register.html`, `member/mypage.html` 등 | `templates/` |
| 2-10 | (마지막) `DataInitializer` — admin / user1 시드 | `config/DataInitializer.java` |

**✅ 확인**

- [ ] 회원가입 후 로그인
- [ ] `admin` / `user1` 시드 로그인 (README 계정)
- [ ] 로그아웃 후 세션 제거
- [ ] `@RequestParam(name = "...")` 등 파라미터 이름 명시 (또는 `build.gradle`에 `-parameters`)

**의존**: 0~1단계

## 3단계 — 추천용 프로필 (UserProfile)

**목표**: 설문 저장 후 추천 엔진이 읽을 수 있는 데이터

| 순서 | 할 일 | 주요 클래스 |
|------|--------|-------------|
| 3-1 | Enum `InteriorStyle`, `Gender` 등 | `domain/` |
| 3-2 | 엔티티 `UserProfile` — Member 1:1 | `domain/UserProfile.java` |
| 3-3 | `UserProfileRepository`, `UserProfileService` | `repository/`, `service/` |
| 3-4 | `MemberController` 또는 전용 — `GET/POST /member/profile` | `web/MemberController.java` |
| 3-5 | `ProfileFormDto`, `templates/member/profile.html` | `dto/`, `templates/` |

**✅ 확인**

- [ ] 로그인 사용자가 평수·예산·스타일 저장
- [ ] DB `user_profiles` 행 생성

**의존**: 2단계 (Member)

**다음 단계와의 관계**: 5단계 추천의 **필수 선행**

## 4단계 — 인테리어 게시판

**목표**: RoomFit의 핵심 콘텐츠 — 글·이미지·조회수·좋아요·댓글

| 순서 | 할 일 | 주요 클래스 |
|------|--------|-------------|
| 4-1 | Enum `PostType`, `PostStatus` | `domain/` |
| 4-2 | `InteriorPost`, `PostImage`, `PostLike`, `Comment` | `domain/` |
| 4-3 | Repository 4종 | `repository/` |
| 4-4 | `FileStorageService` — 이미지 저장·URL | `service/FileStorageService.java` |
| 4-5 | `InteriorPostService` — CRUD, 조회수, 좋아요, 댓글 | `service/InteriorPostService.java` |
| 4-6 | `InteriorController` | `web/InteriorController.java` |
| 4-7 | 화면: `interior/list.html`, `detail.html`, `form.html` | `templates/interior/` |
| 4-8 | 시드: 샘플 인테리어 글 몇 건 | `DataInitializer` 확장 |

**✅ 확인**

- [ ] 목록 `/interior` · 상세 `/interior/{id}`
- [ ] 글쓰기·수정·삭제 (본인 글)
- [ ] 이미지 업로드·표시 (`/uploads/**`)
- [ ] 좋아요·댓글 동작

**의존**: 2단계 (작성자 = Member)

## 5단계 — 맞춤 추천

**목표**: 프로필·협업필터·인기글을 합쳐 결과 화면 표시

| 순서 | 할 일 | 주요 클래스 |
|------|--------|-------------|
| 5-1 | `ScoredPostDto`, `RecommendResultDto` | `dto/` |
| 5-2 | `ColorPaletteTable` — 스타일별 색 팔레트 | `recommend/ColorPaletteTable.java` |
| 5-3 | `RecommendEngine` — 가중치·CF·인기 merge | `recommend/RecommendEngine.java` |
| 5-4 | `RecommendService` — 엔진 호출·DTO 조립 | `service/RecommendService.java` |
| 5-5 | `RecommendController` — `/recommend/result` | `web/RecommendController.java` |
| 5-6 | `templates/recommend/result.html` | `templates/` |

**✅ 확인**

- [ ] 프로필 없으면 안내 또는 기본 추천
- [ ] 프로필 있으면 `/recommend/result` 에 글·색상·배치 조언 표시
- [ ] 설계서 가중치 표와 로직 대응 ([프로젝트-설계서.md §9](./프로젝트-설계서.md))

**의존**: 3단계 (UserProfile), 4단계 (InteriorPost, PostLike)

## 6단계 — 소품 쇼핑

**목표**: 상품 목록·장바구니·찜·리뷰

| 순서 | 할 일 | 주요 클래스 |

| 6-1 | `Product`, `Cart`, `CartItem`, `Wishlist`, `ProductReview` | `domain/` |
| 6-2 | Repository + `ShopService` | `repository/`, `service/ShopService.java` |
| 6-3 | `ShopController` — 목록, 상세, 장바구니, 찜, 리뷰 | `web/ShopController.java` |
| 6-4 | 화면: `shop/list.html`, `detail.html`, `cart.html` 등 | `templates/shop/` |

| 6-6 | 시드 상품 데이터 | `DataInitializer` |

**✅ 확인**

- [ ] `/shop` 상품 목록
- [ ] 장바구니 담기·수량 변경·삭제
- [ ] 찜·리뷰 (로그인 필요)

## 9단계 — 마무리 · 품질

**목표**: 발표·운영에 견딜 수 있는 완성도

| 순서 | 할 일 |
|------|--------|
| 9-1 | `BusinessException`, `ResourceNotFoundException` |
| 9-2 | `GlobalControllerAdvice` 또는 `@ControllerAdvice` — 공통 메시지·에러 페이지 |
| 9-3 | `templates/error/` — 403, 404 등 |
| 9-4 | `build.gradle` — `JavaCompile` `-parameters` |
| 9-5 | README · 설계서 · 데모 시나리오 정리 |
| 9-6 | (선택) MySQL 프로필 `application-mysql.yml` |

**✅ 확인**

- [ ] 잘못된 URL·권한 없음 → 에러 화면
- [ ] 발표 데모: 가입 → 프로필 → 인테리어 글 → 추천 → 쇼핑 → 커뮤니티 → 관리자

## 처음부터 다시 만들 때 vs 완성본 참고하기

| 방식 | 설명 |
|------|------|
| **A. 빈 프로젝트에서 순서대로** | 위 0→9를 그대로 따라 새 모듈에 구현 (학습 효과 최대) |
| **B. 완성본을 단계별로 읽기** | `roomfit` 소스를 0→9 순서로 열어보며 “이 단계에서 추가된 파일”만 diff |
| **C. 기능 단위 복습** | 이미 만든 뒤, 4(인테리어) → 5(추천)만 다시 구현해 보기 |

**B 추천 파일 묶음 (단계별)**

| 단계 | 먼저 볼 패키지/폴더 |
|------|---------------------|
| 2 | `domain/Member.java`, `config/SecurityConfig.java`, `web/AuthController.java` |
| 3 | `domain/UserProfile.java`, `service/UserProfileService.java` |
| 4 | `domain/InteriorPost*.java`, `service/InteriorPostService.java`, `web/InteriorController.java` |
| 5 | `recommend/RecommendEngine.java`, `web/RecommendController.java` |
| 6 | `domain/Product.java`, `service/ShopService.java`, `web/ShopController.java` |
| 7 | `domain/CommunityPost.java`, `service/CommunityService.java` |
| 8 | `service/AdminService.java`, `web/AdminController.java` |

## 자주 막히는 지점

| 증상 | 단계 | 조치 |
|------|------|------|
| `Field 'xxx' doesn't have a default value` | 2~ | DB 컬럼과 `Member` 엔티티 필드 일치, 또는 `@Column` + 기본값 |
| `Name for argument ... not specified` | 2~ | `@RequestParam("keyword")` 명시 또는 `-parameters` |
| 추천 결과 비어 있음 | 5 | 3단계 프로필 저장 여부, 4단계 인테리어 시드 여부 |
| 이미지 안 보임 | 4 | `WebConfig` `/uploads/**`, 업로드 디렉터리 경로 |
| `/admin` 403 | 8 | DB `members.role` = `ADMIN`, 시드 `admin` 사용 |
