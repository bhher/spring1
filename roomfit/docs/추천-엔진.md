# RoomFit 추천 엔진 (RecommendEngine)

> **구현**: `src/main/java/com/example/roomfit/recommend/RecommendEngine.java`  
> **호출**: `RecommendService` → `RecommendEngine.recommend(memberId)`  
> **결과 화면**: `GET /recommend/result` → `templates/recommend/result.html`  
> **점수 상세·예시**: [추천-점수-계산.md](./추천-점수-계산.md)

RoomFit 인테리어·소품 플랫폼에서 **로그인 사용자에게 맞춤형 인테리어 포스트(게시글)** 를 추천하는 핵심 엔진입니다.

단일 방식이 아니라 아래 세 가지를 **하이브리드**로 조합합니다.

| 방식 | 영문 | 이 프로젝트에서의 역할 |
|------|------|------------------------|
| 프로필 기반 | Content-Based | 가입 설문(평수·예산·스타일)과 글 속성 매칭 |
| 협업 필터링 | Collaborative Filtering (CF) | 비슷한 프로필 유저가 좋아요한 글 |
| 인기·트렌드 | Popularity | 조회·좋아요·댓글 + 시간 감쇠 |

---

## 아키텍처 개요

```
RecommendController (/recommend/result)
  └─ RecommendService.getRecommendations(memberId)
       └─ RecommendEngine.recommend(memberId)
            ├─ UserProfileRepository      … 프로필 조회
            ├─ InteriorPostRepository     … 후보·인기글 조회
            ├─ PostLikeRepository         … CF용 좋아요 데이터
            ├─ MemberRepository           … CF 이웃 후보
            └─ ColorPaletteTable          … 스타일별 색상 팔레트
```

**사전 조건**: `UserProfile`이 있어야 합니다.  
Controller에서 프로필이 없으면 `/member/profile`로 redirect하고, Engine 내부에서도 없으면 `IllegalStateException`을 던집니다.

---

## 1. 메인 추천 로직 (`recommend`)

추천 전체 흐름을 제어하는 **진입점**입니다.

### 1-1. 프로필 검증

```java
UserProfile profile = userProfileRepository.findByMemberId(memberId)
    .orElseThrow(() -> new IllegalStateException("추천 프로필을 먼저 등록해 주세요."));
```

3단계(추천 프로필)에서 저장한 설문 데이터를 읽습니다.

### 1-2. 복합 추천 데이터 수집

| 경로 | 메서드 | 건수 | 설명 |
|------|--------|------|------|
| 프로필 맞춤 | `scoreByProfile` | **10건** | 최신 visible 글 50건 후보 → 점수 계산 |
| 인기 트렌드 | Repository 조회 | **5건** | 좋아요·조회·최신순 상위 글 |
| 유사 유저 CF | `recommendBySimilarUsers` | **5건** | 닮은꼴 5명이 좋아요한 글 |

후보 풀:

```java
interiorPostRepository.findTop50ByStatusOrderByCreatedAtDesc(PostStatus.VISIBLE)
```

### 1-3. 결과 결합 및 부가 정보

```java
List<ScoredPostDto> merged = mergeResults(profileBased, cfBased, popular);
List<String> colors = ColorPaletteTable.get(profile.getPreferredStyle());
String layout = buildLayoutAdvice(profile);
```

- **병합**: 프로필 → CF → 인기 순으로 합치고 **중복 제거**, 최종 **15건**
- **색상 팔레트**: 선호 스타일별 HEX 4색 (`ColorPaletteTable`)
- **배치 조언**: 평수·가구·라이프스타일 기반 텍스트 팁 (`buildLayoutAdvice`)

반환 DTO: `RecommendResultDto` (posts, colorPalette, layoutAdvice, preferredStyle)

## 2. 알고리즘 ① — 프로필 기반 점수 (`calculateProfileScore`)

유저 `UserProfile`과 `InteriorPost`를 비교해 **항목별 가산**, 이론상 최대 **100점**입니다.

| 항목 | 조건 | 점수 |
|------|------|------|
| 인테리어 스타일 | 글 스타일 = 선호 스타일 | **+40** |
| 방 크기 | \|글 평수 − 내 평수\| ≤ 2평 | **+25** |
| 방 크기 | \|글 평수 − 내 평수\| ≤ 5평 | **+10** |
| 예산 | 오차율 ≤ 20% | **+20** |
| 예산 | 오차율 ≤ 50% | **+5** |
| 가구 유무 | `hasFurnitureTag` 일치 | **+10** |
| 기본 인기도 | `min(5, log(좋아요수 + 1))` | 최대 **+5** |

`scoreByProfile`은 점수 **> 0** 인 글만 남기고, 점수 내림차순 상위 10건을 선택합니다.

> 평수·예산은 구간이 겹치지 않도록 **if / else if** 로 한쪽만 적용됩니다.

---

## 3. 알고리즘 ② — 협업 필터링 (`recommendBySimilarUsers`)

**"나와 취향이 비슷한 사람들은 어떤 글에 좋아요를 눌렀을까?"** 를 찾습니다.

### 3-1. 프로필 벡터화 (`toVector`)

```
[ 평수/30,  예산/1000,  스타일 ordinal/스타일수,  가구보유(0|1) ]
```

### 3-2. 코사인 유사도 (`cosineSimilarity`)

두 프로필 벡터의 **방향(각도)** 이 얼마나 가까운지 0~1 사이 값으로 계산합니다.

### 3-3. 이웃 선정

- 본인 제외 전체 회원 중 프로필이 있는 유저 대상
- 유사도 **> 0.3** 인 상위 **5명** = 이웃(neighbors)
- 이웃이 없으면 CF 결과는 **빈 목록**

### 3-4. 추천 점수

1. 이웃들이 좋아요한 글 ID 수집
2. **본인이 이미 좋아요한 글 제외**
3. 글별 점수 = Σ (이웃 유사도 × 2) — 해당 글에 좋아요한 이웃만 합산
4. 점수 내림차순 상위 **5건**

서비스 초기에는 좋아요·회원 데이터가 적어 CF 결과가 비거나 적을 수 있고, 이 경우 **프로필 기반 + 인기글**이 결과를 채웁니다.

---

## 4. 알고리즘 ③ — 인기도·시간 감쇠 (`popularityScore`)

단순히 좋아요 많은 **옛 글만** 올라오는 것을 막기 위해 **시간 감쇠(decay)** 를 적용합니다.

```
base  = 조회수 × 0.1 + 좋아요 × 2.0 + 댓글수 × 1.5
decay = 1 / (1 + 경과일 / 7)
점수  = int(base × decay)
```

- **base**: 반응(조회·좋아요·댓글)을 가중 합산
- **decay**: 작성 후 **7일 주기**로 절반 수준까지 감소 → 최신 트렌드 글이 상대적으로 유리

인기글 5건은 Repository에서 미리 뽑고, `mergeResults` 단계에서 `popularityScore`로 점수를 부여합니다.

---

## 5. 결과 병합 (`mergeResults`)

| 순서 | 소스 | 최대 |
|------|------|------|
| 1 | 프로필 기반 (`profileBased`) | 10 |
| 2 | 협업 필터 (`cfBased`) | 5 |
| 3 | 인기글 (`popular`) | 5 |
| **합계** | `LinkedHashSet`으로 ID 중복 제거 | **15** |

먼저 들어온 출처가 우선입니다. 같은 글이 여러 경로에 있으면 **첫 번째 점수**만 유지됩니다.

---

## 6. 부가 기능 — 레이아웃 조언 (`buildLayoutAdvice`)

유저 방 상황에 맞는 **텍스트 인테리어 팁**을 조합합니다.

| 조건 | 조언 예시 |
|------|-----------|
| 방 10평 미만 | 침대 헤드를 창 반대편에, 접이식 테이블·소파베드 검토 |
| 가구 없음 | 침대·책상·수납 3종 먼저 배치 후 소품 추가 |
| 라이프스타일 = `재택` | 책상은 자연광 옆, 모니터는 창과 수직 배치 |
| 위 조건 해당 없음 | 통로 폭 80cm 이상 유지 등 기본 팁 |

여러 조건이 동시에 맞으면 줄바꿈(`\n`)으로 **여러 줄** 반환합니다.

---

## 7. 색상 팔레트 (`ColorPaletteTable`)

선호 `InteriorStyle`마다 HEX 색 4개를 고정 매핑합니다.  
추천 결과 화면에서 **스타일에 어울리는 색 조합** 참고용으로 표시됩니다.

## 8. 이 코드의 장점과 특징

### 하이브리드 모델

- **초기(데이터 적음)**: 프로필 매칭 + 인기글 중심으로도 의미 있는 결과
- **성장(좋아요·회원 증가)**: CF가 살아나며 추천 다양성·정확도 향상

### 성능·운영

- `@Transactional(readOnly = true)` on `recommend` — JPA **읽기 전용** 트랜잭션
- 후보를 최신 50건으로 제한해 스코어링 부담 완화

### 레이어 분리

| 클래스 | 역할 |
|--------|------|
| `RecommendController` | 로그인·프로필 유무, 화면 Model |
| `RecommendService` | Engine 위임 (확장 지점) |
| `RecommendEngine` | 알고리즘·점수·병합 |
| `ScoredPostDto` | 글 + 점수 한 쌍 |
