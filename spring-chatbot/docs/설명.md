# spring-chatbot — 파일별 구조 설명

이 프로젝트는 **REST API로 질문을 받고 → OpenAI Chat Completions API를 호출 → 답변 문자열을 JSON으로 반환**하는 최소 구조입니다.  
아래는 파일을 몇 개씩 묶어 정리한 내용입니다.

## 묶음 1: 프로젝트 뼈대

**파일:** `build.gradle`, `settings.gradle`, `src/main/resources/application.yml`, `SpringChatbotApplication.java`

**역할:** 빌드·의존성, 앱 이름, 서버/외부 API 설정, 부트 실행 진입점.

**파일:** `config/AppConfig.java`, `config/OpenAiProperties.java`

## 묶음 3: HTTP API 계층

**파일:** `api/ChatController.java`, `api/request/ChatRequestDto.java`, `api/response/ChatResponseDto.java`

**역할:** 클라이언트가 호출하는 URL과 요청/응답 형태 정의.

| 엔드포인트 | 설명 |
|------------|------|
| `POST /chat` | JSON `{"message":"..."}` — `@Valid`로 검증 |
| `POST /chat/text` | body 전체를 문자열로 받는 **간단 테스트용** |

컨트롤러는 `ChatService.ask(...)` 결과를 `ChatResponseDto.of(...)`로 감싸 반환합니다.

```11:32:e:\spring1\spring-chatbot\src\main\java\com\example\chatbot\api\ChatController.java
@RestController
public class ChatController {

	private final ChatService chatService;

	public ChatController(ChatService chatService) {
		this.chatService = chatService;
	}

	@PostMapping("/chat")
	public ChatResponseDto chat(@Valid @RequestBody ChatRequestDto req) {
		return ChatResponseDto.of(chatService.ask(req.message()));
	}

	/**
	 * 간단 테스트용: JSON 문자열(body가 "안녕") 또는 text/plain("안녕") 형태로 호출 가능
	 */
	@PostMapping("/chat/text")
	public ChatResponseDto chatText(@RequestBody String userMessage) {
		return ChatResponseDto.of(chatService.ask(userMessage));
	}
}
```

**ChatRequestDto:** record + `@NotBlank String message` — 빈 메시지면 검증 실패 → `GlobalExceptionHandler`에서 400 처리.

**ChatResponseDto:** API 바깥으로 `content` 필드 하나만 노출.

```1:9:e:\spring1\spring-chatbot\src\main\java\com\example\chatbot\api\response\ChatResponseDto.java
package com.example.chatbot.api.response;

public record ChatResponseDto(
		String content
) {
	public static ChatResponseDto of(String content) {
		return new ChatResponseDto(content);
	}
}
```

## 묶음 4: 비즈니스 흐름

**파일:** `service/ChatService.java`

**역할:** API 키 존재 여부 확인, OpenAI 요청 DTO 조립, `OpenAiClient` 호출, 응답에서 답문장만 추출.

- API 키가 비어 있으면 `IllegalStateException` — 메시지에 환경 변수 설정 안내.
- `ChatCompletionsRequest`에 **시스템 메시지**(한국어 답변 유도)와 **유저 메시지**를 넣음.
- `choices`가 비어 있거나 `message`가 없으면 `IllegalStateException`.
- 정상이면 `choices.get(0).message().content()` 반환.

```22:41:e:\spring1\spring-chatbot\src\main\java\com\example\chatbot\service\ChatService.java
	public String ask(String userMessage) {
		if (props.apiKey() == null || props.apiKey().isBlank()) {
			throw new IllegalStateException("OPENAI_API_KEY가 설정되어 있지 않습니다. 환경변수를 설정하세요.");
		}

		ChatCompletionsRequest req = new ChatCompletionsRequest(
				props.model(),
				List.of(
						new ChatCompletionsRequest.Message("system", "You are a helpful assistant. Answer in Korean."),
						new ChatCompletionsRequest.Message("user", userMessage)
				)
		);

		ChatCompletionsResponse res = openAiClient.chatCompletions(req);
		if (res == null || res.choices() == null || res.choices().isEmpty() || res.choices().get(0).message() == null) {
			throw new IllegalStateException("OpenAI 응답이 비어 있습니다.");
		}

		return res.choices().get(0).message().content();
	}
```

## 묶음 5: 외부 HTTP 호출 + OpenAI JSON 모델

**파일:** `openai/OpenAiClient.java`, `openai/dto/ChatCompletionsRequest.java`, `openai/dto/ChatCompletionsResponse.java`

**역할:** Spring `RestClient`로 OpenAI에 POST하고, 요청/응답을 **record DTO**로 직렬화·역직렬화.

- `RestClient.builder().baseUrl(...)`에 공통 헤더 `Content-Type`, `Accept` 설정.
- `chatCompletions`에서 `POST` + `Authorization: Bearer {apiKey}` + JSON body.
- 응답 타입은 `ChatCompletionsResponse.class`로 바인딩.

```26:33:e:\spring1\spring-chatbot\src\main\java\com\example\chatbot\openai\OpenAiClient.java
	public ChatCompletionsResponse chatCompletions(ChatCompletionsRequest request) {
		return restClient.post()
				.uri(props.chatCompletionsPath())
				.header(HttpHeaders.AUTHORIZATION, "Bearer " + props.apiKey())
				.body(request)
				.retrieve()
				.body(ChatCompletionsResponse.class);
	}
```

**ChatCompletionsRequest:** OpenAI가 기대하는 `model`, `messages[{role, content}]` 구조.

**ChatCompletionsResponse:** 실제 API JSON에는 필드가 더 많지만, **필요한 경로만**(`choices[0].message.content`) 역직렬화하면 됨. 나머지 필드는 무시됩니다.

## 묶음 6: 공통 에러 응답

**파일:** `api/error/ApiError.java`, `api/error/GlobalExceptionHandler.java`

**역할:** 예외를 일관된 JSON(`code`, `message`, `timestamp`)으로 변환.

| 예외 | 대략적인 응답 |
|------|----------------|
| `MethodArgumentNotValidException` | `BAD_REQUEST` — 요청 값 검증 실패 |
| `IllegalStateException` | `CONFIG_ERROR` 등 — API 키 미설정, 빈 응답 등 |
| `HttpMessageNotReadableException` | `BAD_REQUEST` — JSON 형식 오류 |
| `HttpClientErrorException` | OpenAI 4xx — 본문 일부 전달 |
| `HttpServerErrorException` | `502` + OpenAI 5xx 본문 |

```12:41:e:\spring1\spring-chatbot\src\main\java\com\example\chatbot\api\error\GlobalExceptionHandler.java
@RestControllerAdvice
public class GlobalExceptionHandler {

	@ExceptionHandler(MethodArgumentNotValidException.class)
	public ResponseEntity<ApiError> validation(MethodArgumentNotValidException e) {
		return ResponseEntity.badRequest().body(ApiError.of("BAD_REQUEST", "요청 값이 올바르지 않습니다."));
	}

	@ExceptionHandler(IllegalStateException.class)
	public ResponseEntity<ApiError> illegalState(IllegalStateException e) {
		return ResponseEntity.status(HttpStatus.BAD_REQUEST).body(ApiError.of("CONFIG_ERROR", e.getMessage()));
	}
	// ... HttpMessageNotReadableException, HttpClientErrorException, HttpServerErrorException
}
```
